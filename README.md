## A Regra

| Dia | Tipo | O que |
|---|---|---|
| Seg, Qua, Sex | **Projeto próprio** | Template, ferramenta, open-source do Data Squad |
| Ter, Qui | **PR externo** | Contribuição em repositório alheio (docs, bugfix, feature) |
| Sáb | **Issue hunting** | Responder issues, revisar PRs, ajudar comunidade |
| Dom | Off / Polish | Refinar README, docs, testes |

> Mínimo viável: **1 commit público por dia**. O conteúdo varia, mas a consistência não.

---

## Por que público?

1. **Portfólio vivo** — qualquer recrutador vê o dia a dia real
2. **Disciplina** — compromisso público é mais difícil de quebrar
3. **Comunidade** — gente que acompanha vira network e oportunidade
4. **Propósito** — cada contribuição resolvendo um problema real de alguém

---

## Como funciona

- **ROADMAP.md** — planejamento semanal com projetos e PRs alvos
- **LOG.md** — registro diário do que foi feito (commit, PR, issue)
- **IDEIAS.md** — lista de ideias para projetos futuros e repositórios alvo
- Cada projeto próprio ganha seu próprio repo no GitHub
- Cada PR externo é linkado no LOG com status
